Segmentação de Clientes - Customer Segmentation¶

Segmentao de clientes para campanhas customizadas de Marketing em uma rede de Food Delivery.

Esse um projeto de clusterizao.

Disponvel tambm em meu github.

O conjunto de dados utilizado est disponvel neste link.

1. Entendimento do Negócio¶

A Big Rango um restaurante do tipo Dark Kitchen, ou seja, um estabelecimento de servio de alimentao que oferece apenas comida para viagem. Atuando em 9 cidades, a Big Rango um restaurante 100% delivery, onde os pedidos so realizados via telefone ou internet, e entregues diretamente aos clientes.

Visando oferecer uma comunicao assertiva e personalizada, a Big Rango contratou um consultor em cincia de dados para realizar a segmentao de seus clientes. O principal objetivo da empresa identificar os principais padres de consumo e assim, realizar campanhas de Marketing customizadas para cada grupo de clientes.

Alm disso, a empresa tambm estuda adicionar novos pratos no cardpio, que atualmente, composto por pizzas, bebidas, saladas e sobremesas. A segmentao dos clientes tambm pode auxiliar a empresa nessa tarefa.

1.1 Dicionário de Dados¶

Em relao ao conjunto de dados, as seguintes informaes foram disponibilizadas:

  • Os dados esto em formato estruturado e sero disponibilizados em um arquivo "csv".
  • A varivel id_transacao possui o mesmo cdigo para cada item dentro de um pedido.
  • Todos os pedidos so referentes ao ano de 2019.
  • A empresa no trabalha das 2 s 10 horas da manh.

Alm disso, tambm foi disponibilizado o dicionrio de dados:

  • id_transacao: identificao do pedido.

  • horario_pedido: horrio do pedido.

  • localidade: local da unidade do restaurante.

  • nome_item: nome do item pedido.

  • quantidade_item: quantidade do item no pedido.

  • latitude: latitude da localidade do restaurante.

  • longitude: longitude da localidade do restaurante.

1.2 Estratégia da Solução¶

Como estratgia para a soluo dos problemas, definimos as seguintes etapas:

  • 1. Entendimento do Negcio: nesta etapa inicial, realizamos uma breve introduo sobre o tema do projeto, entendemos as necessidades do cliente e definimos nossos objetivos.
  • 2. Entendimento dos Dados: o objetivo realizar um tratamento inicial para verificar a qualidade dos dados, tambm realizaremos uma anlise mais detalhada a fim de responder perguntas sobre o negcio.
  • 3. Engenharia de Atributos: nessa sesso, iremos visualizar os dados de uma outra perspectiva, alm de realizar a seleo das melhores variveis.
  • 4. Pr-Processamento dos Dados: nesta etapa, o objetivo preparar os dados para a aplicao do modelo de Machine Learning.
  • 5. Modelagem Preditiva: utilizaremos o K-means, um dos algoritmos mais utilizados para problemas de clusterizao.
  • 6. Anlise de Clusters: iremos explorar os clusters com objetivo de identificar padres e recomendar aes para a rea de Marketing.
  • 7. Concluses Finais: por fim, entregaremos o resultado final do projeto.

p_dia_semana <- function(month, color){
    df1 %>%
    mutate(ano = year(horario_pedido),
           mes = month(horario_pedido),
           dia_mes = day(horario_pedido),
           dia_semana = weekdays(horario_pedido),
           horas = hour(horario_pedido)) %>%
    mutate(dia_semana = factor(dia_semana, levels = c('segunda-feira',
                                                      'tera-feira',
                                                      'quarta-feira',
                                                      'quinta-feira',
                                                      'sexta-feira', 
                                                      'sbado', 
                                                      'domingo'),
                               labels = c('seg', 'ter', 'qua', 'qui', 'sex', 'sb', 'dom'))) %>%
    filter(ano == 2019, mes == month) %>% 
    group_by(dia_semana) %>% 
    dplyr::summarise(n = n()) %>% 
    ggplot(aes(x = dia_semana, y = n)) +
    geom_col(size = 1, fill = color) +
    ggtitle(sprintf('Ms %d', month))+
    xlab('') +
    geom_text(aes(label = round(n/sum(n), 2)), 
              vjust = 1.6, 
              color = 'black', 
              size = 5)}

5. Modelagem Preditiva¶

Um dos principais algoritmos para clusterizao o K-means.

O objetivo desse algoritmo encontrar similaridades entre os dados e agrup-los conforme o nmero de clusters passado pelo parmetro k.

Para mais detalhes sobre esse algoritmo, acesse este link em meu blog de estudos.

5.2 Definindo o Número de Clusters¶

Para essa tarefa, utilizaremos o mtodo de Elbow.

Basicamente o que esse mtodo faz testar a varincia dos dados em relao ao nmero de clusters.

O valor ideal considerado quando o aumento no nmero de clusters no representar um valor significativo de ganho de informao.

In [36]:
# Criando a funo.
elbow <- function(dataset){
    wss <- numeric(15)
    for (i in 1:15) wss[i] <- sum(kmeans(dataset, centers = i, nstart = 100)$withinss)
    plot(1:15, wss, type = 'b', main = 'Mtodo de Elbow', xlab = 'Nmero de Clusters', pch = 8)}

# Itens mais pedidos.
qnt_pedidos <- function(x, cor_escolhida){
    df2 %>% as.data.frame() %>%
    mutate(Cluster = factor(cluster, levels = c(1, 2, 3),
                            labels = c('Cluster 1', 'Cluster 2', 'Cluster 3'))) %>% 
    filter(Cluster == x) %>% 
    select(bebida, pizza, salada, sobremesa) %>% 
    as_tibble() %>% 
    gather(item, qtd, 1:4) %>% 
    group_by(item) %>% 
    dplyr::summarise(qtd = sum(qtd)) %>% 
    ggplot(aes(x = item, y = qtd)) +
    geom_col(fill = cor_escolhida) +
    ylim(c(0, 150000)) +
    ggtitle('Itens Mais Pedidos') +
    geom_text(aes(label = round(qtd/sum(qtd), 2)), 
              vjust = -1, 
              color = 'black', 
              size = 3.5)}


# Distribuio de frequncia.
frq_pedidos <- function(x,y){
    df2 %>% as.data.frame() %>% 
    mutate(Cluster = factor(cluster, levels = c(1, 2, 3),
                            labels = c('Cluster 1', 'Cluster 2', 'Cluster 3'))) %>% 
    filter(Cluster == x) %>% 
    select(bebida, pizza, salada, sobremesa) %>% 
    as_tibble() %>% 
    gather(item, qtd, 1:4) %>% 
    filter(qtd > 0) %>% 
    ggplot(aes(x = qtd)) +
    geom_histogram(aes(y = ..density..), binwidth = 1, show.legend = FALSE, fill = y) +
    facet_grid(~item) +
    ggtitle('Distribuio de Frequncia') +
    xlab('Quantidade')}

# Pedidos por horrio.
hr_pedidos <- function(x){
    df2 %>% as.data.frame() %>% 
    select(horario_pedido, bebida, pizza, salada, sobremesa, cluster) %>% 
    gather(item, qtd, 2:5) %>% 
    mutate(horario = hour(horario_pedido),
           Cluster = factor(cluster,levels = c(1, 2, 3),
                            labels = c('Cluster 1', 'Cluster 2', 'Cluster 3'))) %>% 
    select(Cluster, item, qtd, horario) %>% 
    filter(Cluster == x, qtd > 0) %>% 
    select(item, qtd, horario) %>% 
    as_tibble() %>% 
    ggplot(aes(x = horario, color = item)) +
    geom_freqpoly(binwidth = 1, size = 1) +
    xlim(c(0, 23)) +
    xlab('Hora do dia') +
    ggtitle('Pedidos por Horrio')}

Para a realizao da segmentao dos clientes da Big Rango utilizamos o algoritmo K-means.

Atravs do mtodo de Elbow, definimos que os dados deveriam ser segmentados em trs clusters, foram eles:

  • Cluster 1: clientes mais saudveis; alto consumo de salada.
  • Cluster 2: clientes que fazem pedidos em grandes quantidades; o principal cluster para a empresa.
  • Cluster 3: clientes que fazem pedidos prximos meia noite; o cluster mais equilibrado.

  • Geralmente nos ltimos dias dos meses h uma queda no nmero de pedidos.
